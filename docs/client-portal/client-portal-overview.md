# Client Portal Module

## Purpose

The client portal gives selected customers restricted access to project information, shared documents and approvals.

## Customer Access Rules

Customer users must only see their own account/project data.

They must not see:

- Internal notes.
- Internal budgets.
- Internal resource plans.
- Other customer records.
- CRM pipeline.
- Internal risks unless explicitly shared.

## Portal Features

- View assigned projects.
- View project status.
- View shared milestones.
- Download shared documents.
- Upload requested documents where enabled.
- Approve documents or change requests where enabled.
- See project contact details.

## Portal Views

- My projects.
- Project detail.
- Shared documents.
- Approvals.
- Messages or comments where enabled.

## Acceptance Criteria

The module is acceptable when customer users can securely view only approved project information for their own organisation.
