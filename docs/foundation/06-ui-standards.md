# UI Standards

## Design Style

ConnectedOS should look like a modern enterprise operations platform.

Style keywords:

- Clean.
- Professional.
- Minimal.
- Fast to scan.
- Operationally useful.

## Branding

Primary brand colour should align with Connected Hospitality deep purple.

Do not overuse colour. Use colour primarily for:

- Headers.
- Buttons.
- Status badges.
- Dashboard accents.
- Important alerts.

## Common Layout

The application should include:

- Left navigation sidebar.
- Top bar with search, notifications and user menu.
- Main content area.
- Breadcrumbs where useful.
- Module dashboards.

## Common Views

Where appropriate, modules should provide:

- List view.
- Detail view.
- Board view.
- Calendar view.
- Dashboard view.
- Export option.

## Tables

Tables should support:

- Search.
- Filters.
- Sorting.
- Pagination.
- Column visibility later if useful.
- Row actions.

## Forms

Forms should include:

- Clear labels.
- Required field indicators.
- Validation messages.
- Save and cancel buttons.
- Unsaved changes warning where appropriate.

## Status Badges

Use consistent status badges across modules.

Examples:

- Draft.
- Submitted.
- In Progress.
- Waiting Approval.
- Approved.
- Rejected.
- Complete.
- Overdue.

## Dashboard Cards

Dashboard cards should focus on action:

- What needs attention?
- What is overdue?
- What is high value?
- What changed recently?
- What is at risk?

## Mobile Responsiveness

The platform should be responsive.

Mobile is not the primary working mode for complex admin screens, but users should be able to:

- Review tasks.
- Check schedules.
- Approve expenses.
- Approve leave.
- View dashboards.
- Add notes.
